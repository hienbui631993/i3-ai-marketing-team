## Typography `[confirm]`
- **Headings:** a clean geometric/grotesque sans (e.g. Inter / Montserrat).
- **Body:** a highly legible sans (e.g. Inter / Open Sans).
- Hierarchy: H1 bold, H2 semibold, body regular; generous line height.

## Logo usage `[confirm]`
- Keep clear space around the logo; never stretch or recolor.
- Use the reversed (white) logo on dark/photo backgrounds.
- Add the official logo files to `marketing/templates/` when available.

## Imagery
- Real-world security/operations contexts: retail floors, control rooms, multi-site
  dashboards, cameras in clean modern environments.
- Professional, well-lit, not stocky-cheesy. Avoid threatening/dystopian framing.
- Overlay brand-blue tints sparingly; keep UI/dashboard screenshots crisp.

## Editorial formatting rules
- **Headers** phrased as the reader's question where possible (good for AI search).
- Short paragraphs (2–4 sentences). Bullets over walls of text.
- One clear **CTA** per asset.
- Numbers as numerals ("3 sites", "12% shrink"). Use sentence case for headings `[confirm]`.
- Always include alt-text intent for images in web assets.

## Deck conventions (for Branded Deck skill)
- Title slide → agenda → problem → solution → proof → offer/CTA structure.
- Consistent margins, brand color bar, page numbers, logo on every slide.
- Charts use the brand palette; label clearly; one idea per slide.

## Social conventions
- LinkedIn: hook in line 1, insight body, soft CTA, 3–5 relevant hashtags.
- Carousels: 1 idea per slide, big readable headline, consistent template.
